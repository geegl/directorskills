```yaml
id: CE-01
title: 持续大音量导致听觉疲劳 (Constant Loudness Causes Fatigue)
type: counter-example
source_chapter: 第一章/第三章
source_quote: "避免持续的大音量声音，因为这会导致观众没有听觉放松从而影响效果。关键不在于音量，而在于变化和对比"
summary: >
  错误做法：追求持续大音量以求震撼效果。
  正确做法：在动态段落间留出"喘息空间"。
  乔治·沃特在《珍珠港》中在爆炸前后降低音量，
  让声音在刮擦时上去再下来。持续尖锐声音会伤害
  观众耳朵并导致听觉疲劳。有时导演不在时还要偷偷降电平。
tags: [counter-example, loudness, fatigue, dynamics, bad-practice]
```

```yaml
id: CE-02
title: 声画关联随意更改 (Inconsistent Sound-Image Association)
type: counter-example
source_chapter: 第八章
source_quote: "如果从手头的音响资料库找了一个声音，有可能颠覆一段观众业已形成的精确声画关联"
summary: >
  错误做法：为方便从音效库随便选声音，不考虑已建立的声画关联。
  后果：观众会觉得"不对劲"。案例：电视剧《筹码》中给哈雷摩托车
  配了错误的摩托车资料声，引起大量观众愤怒投诉。
  正确做法：一旦建立声画参考，除非故事需要，必须始终使用同一声音。
tags: [counter-example, consistency, sound-library, audience-expectation]
```

```yaml
id: CE-03
title: 忽视第一遍剧本阅读直接看画面 (Skipping Script Reading)
type: counter-example
source_chapter: 第一章
source_quote: "不要让你那种想看画面的冲动破坏这个独一无二的机会。一旦看过画面，再要回到未受影响的初识阶段几乎不可能"
summary: >
  错误做法：拿到影片后直接看画面做声音设计。
  后果：丧失了用"心耳"自由想象声音的唯一机会，
  被视觉信息主导，声音设计变成画面的被动附属。
  正确做法：先读剧本，一口气读完，标记关键词和声音直觉。
tags: [counter-example, script-reading, workflow, inner-ear, bad-practice]
```

```yaml
id: CE-04
title: 信息过载饱和点 (Information Saturation Overload)
type: counter-example
source_chapter: 第六章/第八章
source_quote: "语言声的语义保持会极大地受到多人说话或其它声画互动的影响。信息过多刚好导致难以理解程度称为信息的饱和点"
summary: >
  错误做法：同时呈现过多声音信息。
  后果：超过"饱和点"后观众无法处理任何信息，
  所有声音都变成噪声。本·伯特指出："你不能听到
  银幕上的所有东西。你要让声音帮助讲故事。"
  正确做法：选择什么不选择什么比堆叠更重要。
tags: [counter-example, saturation, overload, selection, cognitive-limit]
```

```yaml
id: CE-05
title: 音乐过于装饰吸引注意力 (Over-Decorative Music)
type: counter-example
source_chapter: 第七章
source_quote: "如果音乐太复杂或过于装饰，那么音乐很可能会失去情绪表现的功能，从而起不到强化故事的心理效果"
summary: >
  错误做法：使用过于复杂或装饰性的音乐来展示作曲技巧。
  后果：音乐从叙事情节上吸引了注意力，失去情绪功能。
  正确做法：好的电影配乐具有"反讽"——不想被听出来，
  对语言和画面保持从属关系。音乐的目的是联系场景感情、
  引导观众共鸣。
tags: [counter-example, music, decoration, distraction, subordination]
```

```yaml
id: CE-06
title: ADR话筒与同期声不匹配 (ADR Mic Mismatch)
type: counter-example
source_chapter: 第一章
source_quote: "录音棚话筒可能比现场枪式话筒声音'更丰满'，但声音之间的来回切换会使观众出戏"
summary: >
  错误做法：ADR配音时使用与同期录音不同的话筒，
  导致补配音与原声在音色上有明显差异。
  后果：声音来回切换使观众出戏，混录时浪费时间调均衡。
  正确做法：如果只替换一小段，使用与同期录音相同的话筒；
  如果整场替换，可自由选择音色最好的话筒。
tags: [counter-example, adr, microphone, consistency, dialogue]
```

```yaml
id: CE-07
title: 录音时忽视混响 (Recording Without Reverb Control)
type: counter-example
source_chapter: 第一章
source_quote: "要注意任何不需要的混响声；如果你需要的话可以在混录时加上混响声，但你去不掉混响声"
summary: >
  错误做法：在硬墙面空间（浴室/厨房）录音且不处理混响。
  后果：混响一旦录进去就无法去除。
  正确做法：找背景声少的场所录音，注意空调/冰箱/鸟鸣等
  不易察觉的噪声。可以在后期加混响但不能去混响。
  选择夜深人静时录制效果更好。
tags: [counter-example, reverb, recording, environment, acoustics]
```

```yaml
id: CE-08
title: 过度使用资料音乐 (Over-Reliance on Pre-Existing Music)
type: counter-example
source_chapter: 第八章
source_quote: "使用资料音乐可能会因为外部影响而导致在叙事处理当中失去控制。每个观众对资料音乐可能有各自的情绪记忆"
summary: >
  错误做法：大量使用知名歌曲/音乐作为配乐而不考虑叙事一致性。
  后果：每个观众对资料音乐有不同的情绪记忆，
  可能与电影叙事要求背离，导致叙事失控。
  正确做法：谨慎使用，充分考虑文化背景和个人记忆的多样性，
  确保音乐选择与故事主题紧密关联。
tags: [counter-example, pre-existing-music, cultural-memory, narrative-control]
```

```yaml
id: CE-09
title: 忽视声音预算和时间 (Ignoring Sound Budget & Time)
type: counter-example
source_chapter: 第八章
source_quote: "制片厂会在视觉效果上花费3000万美元，但声音预算却没怎么增加。我们为了把事情做得最好，基本上要靠请求、筹借甚至'偷窃'"
summary: >
  错误做法：在视觉效果上花大钱，声音预算严重不足，
  进度表不给声音设计足够时间。
  后果：声音设计师无法充分创造，影片声音质量受限。
  乔治·沃特指出声音从开场到片尾始终存在并对整部影片
  做出贡献，应获得与视觉效果相当的资源。
tags: [counter-example, budget, time, production, industry-issue]
```

```yaml
id: CE-10
title: 期望堆叠导致厌烦 (Expectation Always Met = Boredom)
type: counter-example
source_chapter: 第五章
source_quote: "如果每次预期都非常准确，没有半点变化，那就会让人觉得厌烦、机械"
summary: >
  错误做法：每次声音设计都完全按观众预期行事。
  后果：可预测性导致厌烦和机械感。
  正确做法：在预期的满足和违背之间找到平衡节奏。
  对预期的违背本身也有它的节奏，最好在音乐的节奏偏差、
  旋律偏差、和声偏差上实现平衡。
tags: [counter-example, expectation, predictability, boredom, balance]
```
