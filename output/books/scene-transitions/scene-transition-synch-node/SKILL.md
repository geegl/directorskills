    13|    13|
    14|    14|## R — 原文 (Reading)

    21|    21|## I — 方法论骨架 (Interpretation)
    22|    22|
    23|    23|多线叙事（网络叙事）需要连接点——不同故事线之间的"巧合"让观众理解这些故事线是相关的。同步节点就是这些连接点，可以是：
    24|    24|1. **物体**: 《巴别塔》的来复枪、《撞车》的小雕像——一个物件出现在多条故事线中
    25|    25|2. **场所**: 《大象》的学校走廊——同一地点承载多条故事线
    26|    26|3. **人物**: 《撞车》中两条故事线的人物偶然相遇
    27|    27|
    28|    28|关键洞察：同步节点不是简单的"巧合"——它是让观众在情感层面理解多条故事线相关性的工具。没有同步节点，多线叙事就是一堆互不相关的故事。
    29|    29|
    30|    30|## A1 — 书中的应用 (Past Application)
    31|    31|
    32|    32|### 案例: 《撞车》的多重同步节点
    33|    33|- **问题**: 如何让观众理解洛杉矶不同种族群体的故事是相关的？
    34|    34|- **方法论的使用**: 使用小雕像（物体）、洛杉矶街头（场所）、警察救人（人物相遇）作为同步节点连接多条故事线。
    35|    35|- **结论**: 多个同步节点创造了复杂的故事网络。

## B — 边界 (Boundary) ★

### 不要在以下情况使用此 skill

- 单线叙事不需要同步节点
- 不要在所有场景转换中都使用同步节点——过度使用会让效果减弱

### 作者在书中警告的失败模式

- 同步过于明显变成技巧炫耀——观众注意到技术而非故事
- 同步节点位置不当——在错误的时间点同步会制造混乱

### 作者的盲点

作者假设同步总是增强叙事，但某些反讽叙事可能故意打破同步
