# counter-examples.md — 常见调色错误

```yaml
id: e01
title: "用电脑显示器而非广播监视器做调色决策"
type: counter-example
source_chapter: 第2章
source_quote: "客户有时会指着电脑显示器上的调色界面说：'你能不能让这个图像看起来更像那个？'"
summary: 错误：以电脑显示器上的图像作为调色参考。电脑显示器的色域、伽马、色温与广播监视器不同，且通常未经校准。正确做法：所有调色决策应基于严格校准的广播级监视器，客户和调色师应观看同一台监视器。
tags: [监视器, 显示设备, 校准, 常见错误]
```

```yaml
id: e02
title: "提高黑电平导致暗部浑浊发灰"
type: counter-example
source_chapter: 第3章
source_quote: "如果原始图像有大面积的纯黑位，提高黑电平会产生很平的灰色区域，这样的画面并不好看。"
summary: 错误：在暗部本已很黑的画面中盲目提高黑电平以"显示更多细节"。结果是纯黑区域变成不自然的灰色，破坏画面质感。正确做法：只在暗部确实有丰富细节时才提高黑电平；对于本就应该是纯黑的区域保持黑位不动。
tags: [黑电平, 暗部, 浑浊, 灰色, 常见错误]
```

```yaml
id: e03
title: "过度校正肤色导致'人造感'"
type: counter-example
source_chapter: 第8章
source_quote: "矫正过正是打乱日程安排的一个大问题，如果你不小心，过度调色所导致的'人造感'风险就会加大。"
summary: 错误：对每个镜头的肤色都做二级调色，追求完美匹配I-bar。过度校正会导致肤色看起来不自然，失去妆容效果和现场光照的真实感。正确做法：一级校色后的肤色如果健康自然就不需要额外二级调整；有化妆师参与的项目以维持原始妆容为目标。
tags: [肤色校正, 过度调色, 人造感, 矫正过正]
```

```yaml
id: e04
title: "在静止帧上做二级调色导致播放时出现抖动"
type: counter-example
source_chapter: 第5章
source_quote: "在静止状态下观察画面、制作蒙版和进行二级调色都是极具风险的，有可能前面的工作都会白费。"
summary: 错误：只在静止帧上创建和优化HSL选色蒙版。静止状态看起来完美的蒙版在播放时可能出现边缘抖动、闪烁或颜色溢出。正确做法：始终在播放状态下反复检查蒙版质量，因为各种键控问题都不会反映在静止帧上。
tags: [静止帧, 播放检查, 蒙版抖动, HSL选色]
```

```yaml
id: e05
title: "简单降低Gain来合法化超白导致中间调变暗"
type: counter-example
source_chapter: 第3章
source_quote: "如果只是简单地降低Gain，你会将中间调也压暗。"
summary: 错误：为将超白信号合法化而简单降低Gain，结果中间调也被压暗，画面整体变暗失去活力。正确做法：在压低白位的同时提高Gamma几个点来补偿中间调。在调色台上可同时操作多个控件提高效率。
tags: [广播安全, Gain, 中间调, 合法化, 常见错误]
```

```yaml
id: e06
title: "在高压缩素材上期望获得优质键控"
type: counter-example
source_chapter: 第5章
source_quote: "采用4:1:1和4:2:0色度采样的视频格式来做键控会出现块状边缘，比从4:4:4或者4:2:2素材做键控产生更多的压缩失真。"
summary: 错误：在4:1:1或4:2:0高压缩素材上用色相和饱和度限定控件做精细键控。色度信息不足会导致块状边缘和压缩失真。正确做法：(1)尽量使用高质量素材（4:2:2或4:4:4）；(2)在高压缩素材上只用亮度限定控件做键控（亮度通道保留完整信息）；(3)使用色度平滑插件改善边缘。
tags: [高压缩素材, 键控质量, 色度采样, 4:2:0, 常见错误]
```

```yaml
id: e07
title: "忽略调色间环境光对色彩判断的影响"
type: counter-example
source_chapter: 第2章
source_quote: "如果环绕墙是橙色的，你会不自觉地对图像色彩做出错误的调整。"
summary: 错误：在有彩色墙壁或不当环境光的房间里调色。人眼会根据周围环境来判断色彩，非中性的环境会导致调色师做出错误的色彩补偿。正确做法：环绕墙应为18%中性灰，环境光应匹配目标显示标准（D65/D93），避免直射光打到监视器上。
tags: [调色间, 环境光, 环绕墙, 色彩感知, 灰色]
```

```yaml
id: e08
title: "在一个镜头上耗时过长导致失去客观性"
type: counter-example
source_chapter: 第9章
source_quote: "如果你在一个镜头上花费超过几分钟的时间，会开始感到沮丧，建议你继续工作，前进到另一个镜头。"
summary: 错误：在单个镜头上反复调整数十分钟。人眼的适应性会导致调色师逐渐失去对色彩差异的敏感度，做出越来越偏的调整。正确做法：设定时间限制，遇到困难镜头先跳过继续工作，第二天再回来——"第二天规则"往往能在一两分钟内解决问题。
tags: [工作效率, 眼睛适应性, 客观性, 时间管理]
```
