title: "一级校色三步工作流：对比度→色彩→饱和度"
type: framework
source_chapter: 第3章、第4章
source_quote: "比较好的调整方法是，在最开始调整时先调整画面对比度，然后再调整色彩。颜色和反差调整之间的互动会十分频繁。"
summary: 专业调色的标准工作流是：先用Lift/Gamma/Gain调整对比度使画面有良好的黑位和白位，再用色彩平衡控件校正偏色，最后微调饱和度。对比度和色彩控件之间有强交互性，调整一个会影响另一个，因此需要迭代调整。使用调色台可同时操作多个参数提高效率。
tags: [一级校色, 工作流, 对比度, 色彩平衡, Lift-Gamma-Gain]

id: f02
title: "二级校色工作流：HSL选色→形状遮罩→组合蒙版"
type: framework
source_chapter: 第5章、第6章
source_quote: "二级调色的质量由键控的质量来决定。每个镜头的复杂程度都不同，就算是一个普通的键控，也有能通过选色技巧来提升这个镜头。"
summary: 二级调色的通用工作流：(1)选择隔离方式——HSL选色适合有独特色相/饱和度/亮度特征的物体，形状遮罩适合分离明暗区域；(2)优化键控质量——提高对比度和饱和度有助键控，使用原始素材或调色后素材作键控源需根据情况选择；(3)组合多个蒙版用布尔运算（合并/相减/相交）创建复杂选区。质量由键控边缘的平滑度决定。
tags: [二级校色, HSL选色, 形状遮罩, 蒙版组合, 键控]
```

id: f03
title: "镜头匹配工作流：评估→对比度→色彩→饱和度→异常→迭代"
type: framework
source_chapter: 第9章
source_quote: "镜头匹配也是一个迭代过程。不要害怕来调整对比度和颜色，因为每次的改变都会对另一个镜头产生微妙的影响。"
summary: 场景内镜头匹配的标准流程：(1)播放整个场景评估差异；(2)选择参考镜头并调好对比度；(3)逐一匹配其他镜头的对比度；(4)匹配色彩平衡；(5)检查饱和度是否一致；(6)排查异常干扰元素；(7)从头播放整个场景确认。关键原则：知道何时停止，眼睛适应性会导致失去客观性。
tags: [镜头匹配, 场景平衡, 工作流, 一致性]
```

id: f04
title: "Log编码素材调色流程：Log状态调整→正常化→细节调整"
type: framework
source_chapter: 第3章
source_quote: "对已经正常化的素材使用Shadow、Midtone、Highlight工具，你会发现，每个影调之间的重叠比较轻微，不是很实用。这就是在套用LUT之前，调整这些控件的原因。"
summary: Log编码素材的专业调色流程：(1)在Log编码原始状态下使用Offset/Exposure做整体调整；(2)用Shadow/Midtone/Highlight工具在Log状态下做初步影调分区调整——此时控件影响范围更宽、过渡更平滑；(3)套用LUT或正常化转换；(4)在正常化后用Lift/Gamma/Gain和色彩平衡控件做精细调整。核心逻辑：Log状态适合大刀阔斧，正常化后适合精雕细琢。
tags: [Log编码, LUT, 正常化, 调色流程, 动态范围]
```

title: "创意色彩风格开发流程：评估→一级基础→二级分区→风格叠加"
type: framework
source_chapter: 第7章、第11章
source_quote: "在做出任何调整之前先评估镜头，规划一下调整思路。"
summary: 创意Look开发的通用流程：(1)评估原始素材，规划调整思路；(2)建立扎实的一级校色基础（良好的对比度和中性色彩）；(3)用二级调色分离需要单独处理的区域（肤色/天空/暗部等）；(4)叠加风格化调整（着色/偏色/暗部色调/饱和度处理）。关键原则：先做技术正确的一级校色，再在其上叠加风格化处理，二者分离便于后期修改。
tags: [Look开发, 创意调色, 风格化, 工作流]
```
